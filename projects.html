Flight Aggregator

What it does

This Python project aggregates data from over 67,000 flight routes and 9,300 airports into a single searchable interface. It allows users to explore global flight paths, find direct or connecting routes between cities, and save itineraries on to a CSV file.

Why I built it

Originally a High school assignment, this was my very first large scale programming project. It served as a practical challenge to implement automated pathfinding for connecting flights and for handling large amounts of data.

What I would do differently

I would firstly transition from a CLI to a web-based GUI. Additionally, I would implement advanced pathfinding algorithms to support multi-stop itineraries.

Technology Tradeoffs

I chose to upload the raw CSV and text data into MySQL tables to ensure faster query speeds and more robust relational linking. While this required more backend boilerplate code than a simple file-reader, it significantly improved performance.

Twixt

What it does

This is an implementation of a simple two-player game board game using multifile C programming and linked lists.

Why I built it

This being my first large C project, I wished to gain experience in multifile programming and accurate handling of linked lists to encode state information.

What I would do differently

Firstly, and most obviously, I would rather have built this using object-oriented programming if I could do this today, so as to make it much more efficient. Secondly, I wish I would have put some more effort into the UI of the game.

Technology Tradeoffs

I opted for pure C with manual memory management for the linked lists instead of a higher-level language with garbage collection like Python. This resulted in a significantly faster, more memory-efficient executable, but it required meticulous debugging of pointer logic and memory leaks.

This Website!

What it does

This website serves as a record of me as I would present myself. My skills, projects, and Milestones have been neatly logged for anyone who wishes to quickly evalute my abilities.

Why I built it

While yes, this was a task for a course assignment, I had always wanted a place to log my achievements. There may not be much here at the moment, but it will serve as an invaluable record as my skills grow.

What I would do differently

If I had my way, I would have put all the 4 pages of the website onto a singular large page where everything can be seen at once. Unfortunately, the assignment guidelines forbade it.

Technology Tradeoffs

I utilized HTML, CSS, and JavaScript rather than relying on modern frameworks. This required more boilerplate code and manual state management, but it ensured a zero-dependency, blazing-fast site that complies strictly with the W3C and Lighthouse accessibility constraints.
